# WeReadMac

将[微信读书](https://weread.qq.com)网页版封装为独立的 macOS 应用，解决日常在浏览器中使用时标签页容易被误关闭的问题，同时提供本地笔记管理能力。

## 功能特性

### 原生应用体验

- 独立的 macOS 应用，拥有独立的 Dock 图标和 Cmd+Tab 切换支持
- 基于 WKWebView，完整支持微信读书网页版所有功能（书架、阅读、笔记、搜索等）
- 登录状态持久化，关闭重开无需重复登录
- 窗口位置和大小自动记忆
- 自动跟随系统深色/浅色模式
- 外部链接自动在系统浏览器中打开
- 网络异常时显示友好提示页面，支持一键重试

### 笔记自动采集

通过 JavaScript 拦截微信读书 API，在阅读过程中自动采集笔记数据并持久化到本地 Core Data 数据库：

- **划线（高亮）**：支持直线、马克笔、波浪线三种样式
- **想法（批注）**：用户添加的评论和注解，包含关联原文
- **书签同步**：从服务端同步书签列表

自动获取书籍信息（书名、作者、封面）和章节信息，实现笔记元数据的完整记录。

### 笔记搜索

- 快捷键 Cmd+Shift+N 打开搜索窗口
- 支持在划线文本、想法内容、章节标题、书名中全文搜索
- 按书籍分组展示搜索结果，显示各书命中数量
- 实时搜索，输入即查

### 笔记浏览

- 左侧书籍列表 + 右侧笔记详情的分栏布局
- 按「划线」和「想法」分区展示
- 显示高亮类型标签、章节标题、创建时间等元信息
- 笔记文本可选中复制

### 笔记导出

支持将笔记导出为两种格式：

- **Markdown**：按书籍 → 章节 → 笔记层级组织，包含书名、作者、导出时间、笔记数量统计
- **JSON**：结构化数据格式，包含完整元信息，适合程序化处理

支持导出全部书籍或单本书籍的笔记。

### 笔记删除

- 本地删除：从 Core Data 数据库中移除，删除前弹窗确认
- 远程同步删除：同步删除微信读书服务端的划线和想法数据

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| Cmd+R | 刷新页面 |
| Cmd+[ | 后退 |
| Cmd+] | 前进 |
| Cmd+Shift+N | 搜索笔记 |
| Cmd+, | 打开设置 |

### 设置

- 支持自定义 User-Agent，可随时重置为默认值

## 系统要求

- macOS 13 (Ventura) 或更高版本

## 安装

### 方式一：下载 DMG（推荐）

1. 在 [Releases](../../releases) 页面下载最新的 `WeReadMac.dmg`
2. 双击打开 DMG 文件
3. 将 `WeReadMac.app` 拖入 `Applications` 文件夹
4. 首次打开时，如果系统提示"无法验证开发者"，请前往 **系统设置 → 隐私与安全性** 点击"仍要打开"

### 方式二：从源码编译

```bash
git clone git@github.com:stutiredboy/WeReadMac.git
cd WeReadMac
xcodebuild -project WeReadMac.xcodeproj -scheme WeReadMac -configuration Release build
```

或者使用 Xcode 打开 `WeReadMac.xcodeproj`，选择 Release 配置后 Cmd+R 运行。

## 技术实现

- Swift + SwiftUI (App lifecycle) + WKWebView
- Core Data 本地数据持久化（书籍、划线、想法三个实体）
- JavaScript 注入拦截 XMLHttpRequest 和 Fetch 请求，实现笔记自动采集
- WKScriptMessageHandler 实现 JS ↔ Swift 通信桥接
- 使用系统默认 `WKWebsiteDataStore` 持久化 cookies 和 localStorage
- 零第三方依赖，仅使用 Apple 原生框架
